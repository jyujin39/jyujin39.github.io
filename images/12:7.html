지금까지는 RDBMS 의 하나인 mysql을 했었는데,

NoSQL은 RDBMS의 한계를 좀 더 극복한 데이터베이스이다.

  • 장점:

    • 확장성이 좋다.

      Join 개념이 없다. 테이블을 서로 합치거나 할 수 없다.

    • 데이터 저장이 유연하다

      테이블에 새로운 타입의 데이터를 추가해야 할 때 schema를 수정하거나 할 필요 없이 자동으로 칼럼을 생성해서 추가해준다.

    • insert가 빠르다.

  • 단점:

    • 테이블별로 데이터를 서로 나눠 가지면서 join을 할 수가 없기 때문에 같은 데이터를 여러 테이블에서 중복적으로 가지고 있어야 할 수도 있다. 컬럼개수가 늘어나기 때문에 조회할 때 속도가 늦어진다.
    • select는 RDBMS보다 느리다.

NoSQL은,

table 대신 collection 이라는 개념을 쓴다.

Row 대신 document 라는 개념을 쓴다.

document 안에는 key, value 값이 들어있다.

주로 Mongodb 데이터베이스를 사용한다.

db 삭제: db.dropDatabase()

db.createCollection(name, [option])

option

  • capped : true로 설정하면 collection의 최대 용량을 설정 (최대 용량의 크기는 size 옵션으로
    설정), 설정된 최대용량 이상으로 데이터가 입력되면 오래된 데이터 부터 자동으로 삭제됩니다.
  • autoIndex : true로 설정하면 _id 필드에 index가 자동으로 생성됩니다.
  • size : 숫자 데이터를 사용하며 collection의 최대 사이즈를 byte 단위로 지정
  • max : 숫자 데이터를 사용하며 최대 document 개수를 설정

max가 5로 설정된 info1에 6개의 데이터를 넣어보면 5개밖에 안들어간다.

info2는 같은 5개로 max가 설정됐지만 size가 50바이트로 지정되어 4개까지밖에 안들어간다.
